各模块用途（结合 README）

A · 智能路由网关 Router

多模型路由、租户管理、灰度发布、成本统计。把用户请求导向最合适的 LLM 适配器。

配合：④ Supervisor 拿到规划后，由 Router 选择并调用底层模型；所有模型调用经防呆中间件。

B · 企业 RAG 引擎 rag_agent

多模态文档解析、混合检索（向量+BM25+RRF）、零幻觉抽取式摘要（rag_answer）。

配合：⑤ RAG Worker 注册为工具 → ④ ToolRegistry 分发；检索后 ② 认证中间件做权限硬过滤再交给 LLM。

C · Dify 应用编排 dify_bridge

可视化工作流编排、多模型网关、外部知识库集成；FastAPI 侧提供 /dify/* 工具节点桥。

配合：① Nginx 反代 Dify；③ 后端通过 Bridge 把 FDE 工具暴露给 Dify 工作流。

D · 统一消息枢纽 im_agent

企微/飞书/钉钉多 IM 平台接入、会话管理、消息路由（Pydantic 模型 + 适配器 Stub）。

配合：⑤ IM Worker；把外部对话事件喂给 ④ Supervisor，再把响应推回 IM。

E · 桌面 AI 助手 client_agent

Tauri 桌面客户端 SDK：全局快捷键、文本捕获、AI 回填。

配合：② 前端 SDK；通过 ③ API 调用后端 Agent。

F · 数据情报服务 data_agent

全球/多源数据采集、ETL 清洗、归一化（含字段映射与 CanonicalDocument）。

配合：⑥ 入库 Postgres/Qdrant；为 ⑤ Analysis/RAG 提供结构化 + 向量数据。

G · 智能分析层 analysis_agent

NL2SQL 自然语言转查询、交互式 Dashboard、下钻关联分析（scipy/pandas）。

配合：⑤ Analysis Worker；SQL 经安全校验后查 ⑥ Postgres；结果回 ② 图表。

H · 治理与可观测 governance_agent

统一认证/权限(RBAC+ABAC)、审计、评测(Eval)、全链路追踪。

配合：③ AuthMiddleware 硬过滤；所有请求留 决策链日志；④ 追踪 trace_id 贯穿全链路。

J · 人力资源智能决策 hr_agent

员工画像、人岗匹配、风险评估、裁员评估；5 步防呆校验。

配合：⑤ HR Worker；调用 ⑥ 数据；敏感操作走 ② 防呆 + ③ 权限校验。

K · LangGraph 编排层 orchestrator

Supervisor-Worker 状态机：规划 → 路由 → 执行 → 冲突裁决 → 响应生成。

配合：中枢。接收 ③ 请求，调度 ⑤ 各 Worker，经 ToolRegistry 分发，结果聚合回 ②。

L · 地图 AI 分析 map_agent

实体标记、分析收纳盒、语音输入、相关性分析、可视化（ECharts 热力/散点）+ 地图联动。

配合：② MapAI 前端发起；调 ⑤ Map Worker → 后端 /map/analysis 流水线（取实体→相关性→AI 解读）。

连接器 Agent connector_agent

（规划中）对接外部业务系统（用友/金蝶/WMS/TMS 等 Java 连接器），统一 /manifest 契约 + 适配器入库。

配合：消费 Java 连接器 REST；归一为 CanonicalDocument → ⑥ 入库 → ⑤ RAG/Analysis 复用。

共享层 shared/

SDK（trace/decorators/otel）、Models、Prompts、Utils（config/ids/hashing/retry/validators）。

配合：被所有 ③⑤ 模块复用；TraceContext 线程安全、OTel 可插拔后端。

前端即插即用 MapAIModule

prop 驱动 + 事件发射 + UMD/iframe 双集成；数据解耦(static/remote/backend)。

配合：① 第三方看板通过 iframe/UMD 嵌入；运行时指定地图 provider/AK。

核心设计原则（README §1.2）
模块化：各模块松耦合，可独立安装/升级/替换。
本地优先：PII/私域数据组件必须本地部署。
LLM 只规划：Main Agent 输出结构化指令，工具调用由后端执行。
权限硬过滤：权限在后端实现，不经过 LLM，杜绝提示词注入。
零幻觉：合规场景强制溯源引用，相似度低于阈值直接拒答。
防呆设计：所有操作含确认/校验/回退（HR 5 步、SQL 安全校验、地图空实体校验）。
